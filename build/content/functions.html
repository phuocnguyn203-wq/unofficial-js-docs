Language Basics

Functions

Functions are reusable blocks of code — and in JavaScript, they're also values you can store, pass around, and return, just like a number or a string. That second part is one of the most important ideas in the whole language.

Function declarations

[[code lang="js"]] function greet(name) { return `Hello, ${name}!`; } greet("Ava"); // "Hello, Ava!" [[/code]]

A function declaration is hoisted — you can call it before its definition appears in the file, because the engine registers the whole function before running any code. (Full explanation in Scope & Hoisting.)

[[code lang="js"]] sayHi(); // works, even though sayHi is defined below function sayHi() { console.log("Hi!"); } [[/code]]

Function expressions

You can also store a function in a variable. Unlike a declaration, a function expression is not hoisted in a usable way — the variable exists, but isn't assigned the function yet until that line runs.

[[code lang="js"]] const greet = function (name) { return `Hello, ${name}!`; }; greet("Ava"); // "Hello, Ava!" [[/code]]

Arrow functions

Introduced in ES6, arrow functions are a more compact syntax for function expressions — and they handle the this keyword differently, which matters a lot in practice (see The this Keyword).

[[code lang="js"]] // Full form const add = (a, b) => { return a + b; }; // Implicit return — no braces means the expression is the return value const add2 = (a, b) => a + b; // A single parameter doesn't need parentheses const double = n => n * 2; // No parameters still need empty parentheses const sayHi = () => console.log("Hi!"); [[/code]]

Note

Arrow functions are extremely common for short callbacks (like inside .map() or .filter() — see Arrays & Array Methods), but they aren't a full replacement for regular functions everywhere. They don't get their own this, can't be used as constructors, and don't have an arguments object.

Parameters & arguments

Default parameters

Give a parameter a fallback value used when the caller doesn't provide one (or passes undefined):

[[code lang="js"]] function greet(name = "friend") { return `Hello, ${name}!`; } greet(); // "Hello, friend!" greet("Ava"); // "Hello, Ava!" [[/code]]

Rest parameters

Collect any number of remaining arguments into a real array using ...:

[[code lang="js"]] function sum(...numbers) { return numbers.reduce((total, n) => total + n, 0); } sum(1, 2, 3); // 6 sum(10, 20, 30, 40); // 100 [[/code]]

Rest parameters must come last, and there can only be one. (This same ... syntax is also used for spreading values — the opposite operation — covered in Destructuring, Spread & Rest.)

Return values

A function without an explicit return returns undefined. return also exits the function immediately — any code after it in the same block never runs.

[[code lang="js"]] function isEven(n) { if (n % 2 === 0) { return true; } return false; } function noReturn() { console.log("did something"); } console.log(noReturn()); // logs "did something", then undefined [[/code]]

Functions as values

This is the idea that makes JavaScript feel different from many other languages: a function is a value, so you can pass one function into another, or return a function from a function.

[[code lang="js"]] function applyTwice(fn, value) { return fn(fn(value)); } const double = n => n * 2; applyTwice(double, 3); // 12 -> double(double(3)) [[/code]]

A function that takes another function as an argument (or returns one) is called a higher-order function. You've likely already used one without the label — every array method that takes a callback, like .map() or .filter(), is a higher-order function. This idea comes back in force in Functional Programming.

Anonymous vs named functions

A function expression doesn't need a name — it's "anonymous." Naming it anyway helps with debugging, since the name shows up in stack traces:

[[code lang="js"]] const handlers = { onClick: function handleClick() { // named, even as an expression /* ... */ }, }; [[/code]]

Key Takeaways

  • Function declarations are hoisted (callable before their definition); function expressions and arrow functions are not.
  • Arrow functions are compact and don't bind their own this — great for callbacks, not a universal replacement.
  • Default parameters supply fallback values; rest parameters (...args) gather extra arguments into an array.
  • Functions are values in JavaScript — they can be stored, passed as arguments, and returned from other functions (higher-order functions).
