Strings & Template Literals

Strings represent text, and they're immutable — no method ever changes a string in place, they all return a brand new one. Template literals, added in ES6, are now the default way to build strings out of variables.

Creating strings

[[code lang="js"]] const single = 'Hello'; const double = "Hello"; const template = `Hello`; [[/code]]

Single and double quotes are functionally identical — pick one convention and stick with it. Backticks are different: they enable template literals.

Template literals

Template literals support embedded expressions with ${ }, and can span multiple lines without special characters.

[[code lang="js"]] const name = "Ava"; const age = 29; // Old way, with concatenation "Hello, " + name + "! You are " + age + " years old."; // Template literal — cleaner, and evaluates any expression inside ${} `Hello, ${name}! You are ${age} years old.`; `Next year you'll be ${age + 1}.`; [[/code]] [[code lang="js"]] const message = `Dear ${name}, Thank you for your order. We'll see you soon!`; // preserves the line breaks exactly as written [[/code]]

Strings are immutable

Every string method returns a new string — none of them modify the original.

[[code lang="js"]] const greeting = "hello"; greeting.toUpperCase(); // "HELLO" — a new string greeting; // "hello" — completely unchanged [[/code]]

Strings are array-like

You can index into a string and loop over it, but you can't assign to an index (strings are immutable) or use array mutation methods on it directly.

[[code lang="js"]] const word = "cat"; word[0]; // "c" word.length; // 3 word[0] = "b"; // silently does nothing — strings can't be mutated for (const char of word) { console.log(char); // "c", "a", "t" } [[/code]]

Comparing strings

Strings compare lexicographically (character code by character code) with </>, and with strict equality using ===:

[[code lang="js"]] "apple" === "apple"; // true "apple" < "banana"; // true — 'a' comes before 'b' "Apple" === "apple"; // false — comparisons are case-sensitive [[/code]]

Tip

To compare strings while ignoring case, normalize both sides first: a.toLowerCase() === b.toLowerCase().

Key Takeaways

  • Template literals (backticks) support ${expression} interpolation and multi-line strings — prefer them over concatenation.
  • Strings are immutable: every method returns a new string rather than modifying the original.
  • .slice(), .split(), .trim(), and .includes() cover the vast majority of everyday string work.
  • Strings can be indexed and looped over like arrays, but can't be mutated through an index.
