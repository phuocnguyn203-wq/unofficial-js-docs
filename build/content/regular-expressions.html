Regular Expressions

A regular expression (regex) is a pattern for matching text — validating an email's shape, finding every number in a string, replacing all occurrences of a word. This page covers enough to read and write the regexes you'll actually run into.

Creating a regex

[[code lang="js"]] const pattern1 = /hello/; // literal syntax — most common const pattern2 = new RegExp("hello"); // constructor — useful when built from a variable const dynamic = new RegExp(userInput); // only real use case for the constructor form [[/code]]

The building blocks

PatternMatches
\dAny digit (0-9)
\wAny word character (letters, digits, underscore)
\sAny whitespace (space, tab, newline)
.Any single character except a newline
^The start of the string (or line, with the m flag)
$The end of the string (or line, with the m flag)
[abc]Any one of a, b, or c
(abc)A group — also captures the match for later use

Quantifiers control how many times the preceding thing can repeat:

QuantifierMeans
*Zero or more
+One or more
?Zero or one (optional)
{2,4}Between 2 and 4 times
[[code lang="js"]] /\d+/; // one or more digits, e.g. "42" or "7" /colou?r/; // matches "color" AND "colour" — the u is optional /^\d{3}-\d{4}$/; // exactly a 3-digit dash 4-digit pattern, start to end [[/code]]

Flags

FlagEffect
gGlobal — find every match, not just the first.
iCase-insensitive.
mMultiline — ^/$ match the start/end of each line, not just the whole string.
[[code lang="js"]] /hello/i.test("HELLO there"); // true — case-insensitive [[/code]]

Using a regex with strings

[[code lang="js"]] const pattern = /\d+/; pattern.test("Room 42"); // true — does it match at all? "Room 42".match(pattern); // ["42"] — the matched text, as an array-like result "Room 42, Bed 7".match(/\d+/g); // ["42", "7"] — every match, with the g flag "Hello World".replace(/o/g, "0"); // "Hell0 W0rld" — replace every match "a1 b2 c3".split(/\d/); // ["a", " b", " c", ""] — split on a pattern, not just a string [[/code]]

Capture groups

Parentheses ( ) don't just group a pattern — they capture that piece of the match separately, which is especially useful with replace:

[[code lang="js"]] const date = "2026-07-19"; const match = date.match(/(\d{4})-(\d{2})-(\d{2})/); match[1]; // "2026" — year match[2]; // "07" — month match[3]; // "19" — day // Reformat using captured groups in the replacement string ($1, $2, $3) date.replace(/(\d{4})-(\d{2})-(\d{2})/, "$2/$3/$1"); // "07/19/2026" [[/code]]

A couple of practical patterns

[[code lang="js"]] // Loose email shape check — good enough for a quick client-side hint, // NOT a substitute for real server-side validation /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test("ava@example.com"); // true // Extract every number from a string of text "I have 3 cats and 12 fish".match(/\d+/g); // ["3", "12"] // Strip everything except letters and spaces "Hello, World! 123".replace(/[^a-zA-Z\s]/g, ""); // "Hello World " [[/code]]

Tip

Regex gets hard to read fast. For anything beyond a simple pattern, write a comment above it explaining what it's meant to match — future you (and everyone else) will thank you. And when a pattern feels too complex to write by hand, it's fine to look one up rather than reinvent it.

Key Takeaways

  • A regex is a text pattern built from character classes (\d, \w, \s), quantifiers (*, +, ?, {n,m}), and anchors (^, $).
  • .test() checks for a match; .match()/.matchAll() retrieve matches; .replace() substitutes them.
  • The g, i, and m flags control global matching, case sensitivity, and multiline anchors.
  • Capture groups ( ) let you pull out and reuse specific pieces of a match, including inside a replacement string.
