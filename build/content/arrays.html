Working with Data

Arrays & Array Methods

Arrays are JavaScript's ordered lists — and one of the most reached-for data structures in the language. Under the hood an array is just an object with numeric keys and a self-updating length, but in practice, you'll live almost entirely in its rich set of built-in methods.

Creating and accessing arrays

[[code lang="js"]] const fruits = ["apple", "banana", "cherry"]; fruits[0]; // "apple" fruits[fruits.length - 1]; // "cherry" — last item fruits.length; // 3 fruits[3] = "date"; // arrays can grow simply by assigning past the end [[/code]]

Mutating methods

These methods change the array in place, rather than returning a new one.

MethodWhat it does
push(item)Adds to the end, returns the new length.
pop()Removes and returns the last item.
unshift(item)Adds to the start, returns the new length.
shift()Removes and returns the first item.
splice(start, count, ...items)Removes/inserts items anywhere in the array.
sort(compareFn)Sorts in place (see gotcha below).
reverse()Reverses the array in place.
[[code lang="js"]] const nums = [3, 1, 2]; nums.push(4); // nums is now [3, 1, 2, 4] nums.pop(); // returns 4; nums is now [3, 1, 2] nums.splice(1, 1); // removes 1 item starting at index 1; nums is now [3, 2] nums.splice(1, 0, 9); // inserts 9 at index 1; nums is now [3, 9, 2] [[/code]]

Warning

array.sort() converts items to strings and sorts alphabetically by default — [10, 2, 1].sort() gives [1, 10, 2], not [1, 2, 10]. For numbers, always pass a compare function: array.sort((a, b) => a - b).

Transformation methods: map, filter, reduce

These three methods are the workhorses of everyday JavaScript. None of them mutate the original array — each returns something new, which makes code easier to reason about.

map — transform each item

map() returns a new array the same length as the original, with every item passed through a function.

[[code lang="js"]] const prices = [10, 20, 30]; const withTax = prices.map(price => price * 1.1); // [11, 22, 33] — prices itself is unchanged [[/code]]

filter — keep some items

filter() returns a new array containing only the items where the callback returned truthy.

[[code lang="js"]] const numbers = [1, 2, 3, 4, 5, 6]; const evens = numbers.filter(n => n % 2 === 0); // [2, 4, 6] [[/code]]

reduce — combine into a single value

reduce() is the most flexible (and, at first, most confusing) of the three: it walks through the array building up one accumulated value.

[[code lang="js"]] const cart = [10, 20, 30]; const total = cart.reduce((sum, price) => sum + price, 0); // sum starts at 0 (the second argument), then becomes 10, 30, 60 // total === 60 [[/code]]

Think of reduce's callback as running once per item, each time receiving the running total so far (the accumulator) and the current item, and returning the new accumulator for the next round. It can build anything — a sum, an object, even another array:

[[code lang="js"]] const words = ["a", "b", "c"]; const counts = words.reduce((acc, word) => { acc[word] = (acc[word] || 0) + 1; return acc; }, {}); // { a: 1, b: 1, c: 1 } [[/code]]

These three are commonly chained together:

[[code lang="js"]] const orders = [ { item: "Book", price: 15, paid: true }, { item: "Pen", price: 2, paid: false }, { item: "Laptop", price: 900, paid: true }, ]; const totalPaid = orders .filter(order => order.paid) .map(order => order.price) .reduce((sum, price) => sum + price, 0); // 915 [[/code]]

Finding & testing items

MethodReturns
find(fn)The first matching item, or undefined.
findIndex(fn)The index of the first match, or -1.
includes(value)true/false — is this exact value present?
some(fn)true if any item matches.
every(fn)true if all items match.
[[code lang="js"]] const users = [ { id: 1, name: "Ava", admin: true }, { id: 2, name: "Noah", admin: false }, ]; users.find(u => u.id === 2); // { id: 2, name: "Noah", admin: false } users.some(u => u.admin); // true — at least one admin users.every(u => u.admin); // false — not all are admins [3, 6, 9].includes(6); // true [[/code]]

Reading without mutating: slice, concat, join

[[code lang="js"]] const letters = ["a", "b", "c", "d"]; letters.slice(1, 3); // ["b", "c"] — copies a range, doesn't touch letters letters.concat(["e"]); // ["a", "b", "c", "d", "e"] — returns a new, combined array letters.join("-"); // "a-b-c-d" — turns the array into a string [[/code]]

Note

slice (no mutation, takes a range) is easy to confuse with splice (mutates, inserts/removes). If you're not sure which you used, check whether the original array changed.

Iterating an array

Besides for...of (see Control Flow), arrays have their own forEach:

[[code lang="js"]] ["a", "b", "c"].forEach((item, index) => { console.log(index, item); }); // 0 "a" // 1 "b" // 2 "c" [[/code]]

forEach always returns undefined — it's for side effects (like logging), not for building a new value. If you find yourself pushing into an outside array inside a forEach, that's usually a sign map or filter is the better tool.

Checking for an array

typeof reports arrays as "object" (see Variables & Data Types), so use Array.isArray() instead:

[[code lang="js"]] Array.isArray([1, 2, 3]); // true Array.isArray("hello"); // false [[/code]]

Key Takeaways

  • map transforms every item into a new array; filter keeps only matching items; reduce combines everything into one value.
  • Mutating methods (push, splice, sort) change the array in place; slice, concat, map, and filter return new arrays and leave the original alone.
  • Always pass a compare function to sort() when sorting numbers.
  • Use Array.isArray(), not typeof, to check if a value is an array.
