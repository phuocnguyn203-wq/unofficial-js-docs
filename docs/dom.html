Browser & Web APIs

The DOM

The DOM (Document Object Model) is the browser's live, in-memory representation of your HTML page — and it's what JavaScript actually manipulates when it changes what a user sees. This page is where "the language" meets "the browser."

The DOM is a tree#

The browser parses your HTML and builds a tree of node objects — one for the document itself, one for each element, and more for text and attributes. Every element node has a parent, and can have children, mirroring the nesting of your HTML.

HTML
<body>
  <h1>Title</h1>
  <p>Some <strong>text</strong>.</p>
</body>

...becomes a tree: body has two children, h1 and p; p itself has two children, a text node and a strong element. JavaScript's job is to read and rewrite this tree — the browser re-renders the page automatically whenever it changes.

Selecting elements#

MethodReturns
document.getElementById("id")A single element, or null.
document.querySelector("css-selector")The first matching element, or null.
document.querySelectorAll("css-selector")A static list (NodeList) of every match.
JavaScript
const title = document.getElementById("main-title");
const firstButton = document.querySelector(".btn");
const allButtons = document.querySelectorAll(".btn");

allButtons.forEach(btn => console.log(btn.textContent));

Tip

querySelector/querySelectorAll accept any real CSS selector — ".card > h2", "#form input[type='email']", and so on. Learn CSS selectors once and reuse that knowledge here.

Reading and changing content#

JavaScript
const el = document.querySelector("#greeting");

el.textContent;             // reads all text inside, ignoring any HTML tags
el.textContent = "Hello!";  // replaces the content with plain text (safe)

el.innerHTML;                // reads the content AS an HTML string
el.innerHTML = "<b>Hi!</b>"; // parses the string and inserts real elements

Danger

Never set innerHTML using text that came from a user or an external source without sanitizing it first — it will parse and execute any <script> or event-handler attributes it contains. This is the most common source of XSS (Cross-Site Scripting) vulnerabilities. If you're just inserting text, use textContent instead — it's both safer and usually what you meant.

Changing styles and classes#

JavaScript
const box = document.querySelector(".box");

box.style.backgroundColor = "tomato"; // inline style — fine for one-off tweaks
box.style.display = "none";

box.classList.add("active");     // preferred: toggle CSS classes, not inline styles
box.classList.remove("hidden");
box.classList.toggle("open");     // adds it if absent, removes it if present
box.classList.contains("active"); // true

Prefer classList over direct style edits for anything beyond a quick one-off change — it keeps your visual states defined in CSS, where they're easier to see and maintain.

Creating and removing elements#

JavaScript
const list = document.querySelector("#todo-list");

const item = document.createElement("li");
item.textContent = "Buy milk";
list.append(item); // adds it as the last child

item.remove(); // removes it from the DOM entirely

Traversing the tree#

JavaScript
const item = document.querySelector("li");

item.parentElement;       // the <ul> or <ol> containing it
item.children;             // its own child elements
item.nextElementSibling;   // the next <li>, if any
item.previousElementSibling;

Attributes and data-* attributes#

JavaScript
const link = document.querySelector("a");

link.getAttribute("href");
link.setAttribute("href", "/new-page");

// data-* attributes are made for storing custom data on an element
// <button data-user-id="42">Delete</button>
const button = document.querySelector("button");
button.dataset.userId; // "42" — camelCased automatically from "data-user-id"

Key Takeaways

  • The DOM is a live tree of node objects the browser builds from your HTML; JavaScript reads and rewrites it to change the page.
  • querySelector/querySelectorAll accept full CSS selectors and are the most flexible way to find elements.
  • Use textContent for plain text and classList for styling; avoid innerHTML with untrusted input — it's a common XSS vector.
  • createElement + append build new elements; .remove() takes them back out.
