Object-Oriented JS

Classes

ES6 introduced the class keyword as a cleaner, more familiar syntax for the constructor-function-plus-prototype pattern from the previous chapter. If you've used classes in another language, most of this will look immediately familiar — with a few JavaScript-specific twists.

Defining a class#

JavaScript
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  greet() {
    console.log(`Hi, I'm ${this.name} and I'm ${this.age}.`);
  }
}

const ava = new Person("Ava", 29);
ava.greet(); // "Hi, I'm Ava and I'm 29."

The constructor method runs automatically when you call new Person(...) — it's where you set up the instance's own properties. Every other method (like greet) is added to Person.prototype, shared by every instance, exactly as in the previous chapter.

Inheritance with extends and super#

extends creates a subclass that inherits everything from a parent class. Inside a subclass's constructor, super(...) calls the parent's constructor and must run before you can use this.

JavaScript
class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    console.log(`${this.name} makes a sound.`);
  }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name); // must be called before using `this` below
    this.breed = breed;
  }
  speak() {
    super.speak();               // call the parent's version too
    console.log(`${this.name} barks.`);
  }
}

const rex = new Dog("Rex", "Labrador");
rex.speak();
// "Rex makes a sound."
// "Rex barks."

rex instanceof Dog;    // true
rex instanceof Animal; // true — Dog's prototype chain includes Animal

Static members#

static methods and properties belong to the class itself, not to individual instances — useful for utility functions related to the class, or factory methods.

JavaScript
class Circle {
  static PI = 3.14159;

  static fromDiameter(diameter) {
    return new Circle(diameter / 2);
  }

  constructor(radius) {
    this.radius = radius;
  }

  area() {
    return Circle.PI * this.radius ** 2;
  }
}

Circle.PI;                    // 3.14159 — accessed on the class, not an instance
const c = Circle.fromDiameter(10);
c.area();                      // 78.53975

Getters and setters#

Define methods that act like properties — useful for computed values or validation on write.

JavaScript
class Rectangle {
  constructor(width, height) {
    this.width = width;
    this.height = height;
  }

  get area() {
    return this.width * this.height;
  }

  set width(value) {
    if (value <= 0) throw new Error("Width must be positive");
    this._width = value;
  }
  get width() {
    return this._width;
  }
}

const rect = new Rectangle(4, 5);
rect.area;   // 20 — called like a property, no parentheses

Private fields#

Fields prefixed with # are truly private — accessible only from inside the class body, not from outside code, and not even visible via Object.keys(). This is a real language feature (unlike the closure-based privacy from the earlier Closures chapter).

JavaScript
class BankAccount {
  #balance; // private field declaration

  constructor(initialBalance) {
    this.#balance = initialBalance;
  }

  deposit(amount) {
    this.#balance += amount;
    return this.#balance;
  }

  get balance() {
    return this.#balance;
  }
}

const account = new BankAccount(100);
account.deposit(50);
account.balance;    // 150
account.#balance;   // SyntaxError — #balance isn't accessible outside the class

Classes are always strict mode, and never hoisted like functions#

Class bodies run in strict mode automatically, and unlike function declarations, classes are not usable before their declaration line (they sit in the temporal dead zone, same as let/const — see Scope & Hoisting).

JavaScript
new Person("Ava"); // ReferenceError: Cannot access 'Person' before initialization

class Person {
  constructor(name) { this.name = name; }
}

Tip

Everything here is still built on prototypes underneath. If super, instanceof, or method sharing ever feels surprising, re-reading Prototypes & Inheritance usually clears it up — class doesn't add a new object model, just a cleaner syntax for the same one.

Key Takeaways

  • class defines a constructor for instance setup, plus methods that are automatically shared via the prototype.
  • extends sets up inheritance; super(...) calls the parent constructor and must run before this is used.
  • static members belong to the class itself; getters/setters let methods act like properties.
  • Fields prefixed with # are genuinely private — inaccessible from outside the class, unlike regular properties.
