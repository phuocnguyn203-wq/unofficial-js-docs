Introduction

Before you write a single line of code, it helps to know what JavaScript actually is, where it runs, and why it behaves the way it does. This page gives you that big picture — every later chapter builds on it.

What is JavaScript?#

JavaScript is a programming language originally built to make web pages interactive. It was created in 1995 by Brendan Eich at Netscape — reportedly in about ten days — and has since grown into one of the most widely used programming languages in the world.

Today JavaScript is no longer just "the scripting language in web pages." It runs:

  • In the browser, where it can change what's on the page, react to clicks, talk to servers, and build entire applications (Gmail, Figma, and Google Docs are all JavaScript apps at their core).
  • On servers, via runtimes like Node.js or Deno, which let you use JavaScript to build APIs, command-line tools, and backend services.
  • Almost everywhere else — mobile apps (React Native), desktop apps (Electron), even on microcontrollers.

This guide focuses on the language itself and how it's used in the browser, since that's where nearly every JavaScript developer starts.

Note

JavaScript has almost nothing to do with Java. The name was a marketing decision by Netscape in the 90s, when Java was popular. They are unrelated languages with different design goals.

JavaScript and ECMAScript#

The language itself is standardized under the name ECMAScript (ES). "JavaScript" is the popular name for implementations of that standard. You'll see version names like ES5 and ES6 (also called ES2015) thrown around a lot — here's what they mean in practice:

VersionYearWhy it matters
ES52009The baseline nearly every browser supports: Array.prototype.map, strict mode, JSON support.
ES6 / ES20152015The big one: let/const, classes, arrow functions, promises, modules, template literals.
ES2016–ES2026yearlySmaller, focused additions each year — async/await, optional chaining, top-level await, and more.

You don't need to memorize version numbers. This guide teaches modern JavaScript as one continuous language and calls out which features are relatively new when it matters.

How JavaScript actually runs#

When you write JavaScript, you're not writing machine code directly — you're writing instructions for a JavaScript engine: a program that reads your code and executes it. Every browser ships one:

  • V8 — Chrome, Edge, and Node.js
  • SpiderMonkey — Firefox
  • JavaScriptCore — Safari

These engines use Just-In-Time (JIT) compilation: they compile your code to machine code right before running it, rather than ahead of time like C++ or purely interpreting it line-by-line like an old-school scripting language. That's why modern JavaScript is fast enough to build entire applications with — it's a hybrid of "interpreted" and "compiled."

Every engine also gives JavaScript a single call stack — it does one thing at a time — plus extra machinery (a task queue, an event loop) to handle things that take time, like network requests, without freezing everything else. That model is central enough that it gets its own chapter later: The Event Loop.

What makes JavaScript, JavaScript#

A handful of design decisions show up again and again throughout this guide. Knowing them now will make later chapters click faster:

  • Dynamically typed. A variable can hold a number today and a string tomorrow — the type lives on the value, not the variable. See Variables & Data Types.
  • Functions are first-class values. You can store a function in a variable, pass it into another function, or return it from one — this single idea powers callbacks, promises, and most of modern JS. See Functions.
  • Objects use prototypes, not classical classes. Even the class keyword is a friendlier syntax over this underlying model. See Prototypes & Inheritance.
  • Single-threaded, but asynchronous. JavaScript runs one instruction at a time, yet handles timers, network calls, and user input without blocking, using the event loop.

Tip

Don't worry about fully understanding all four ideas right now — they're previews. Each one gets a full, dedicated explanation later in this guide.

Key Takeaways

  • JavaScript began as a browser scripting language and is now used on servers, mobile, and desktop too.
  • "ECMAScript" is the language standard; "JavaScript" is its most common implementation.
  • A JavaScript engine (V8, SpiderMonkey, JavaScriptCore) JIT-compiles your code — a hybrid between interpreted and compiled execution.
  • JavaScript is dynamically typed, treats functions as values, uses prototypes for inheritance, and runs on a single thread with an event loop for async work.
