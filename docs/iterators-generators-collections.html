Modern JavaScript (ES6+)

Iterators, Generators & Collections

This page covers a handful of ES6 additions that don't always get equal billing but show up throughout modern codebases: what actually makes something work with for...of, a compact way to write custom iteration logic, and two collection types that solve real limitations of plain objects and arrays.

What makes something "iterable"#

You've used for...of on arrays and strings since early in this guide (see Control Flow). It works because both types implement the iterable protocol — they have a special method (keyed by Symbol.iterator) that produces an iterator: an object with a .next() method that hands back one value at a time.

JavaScript
const iterator = [10, 20, 30][Symbol.iterator]();

iterator.next(); // { value: 10, done: false }
iterator.next(); // { value: 20, done: false }
iterator.next(); // { value: 30, done: false }
iterator.next(); // { value: undefined, done: true }

for...of, spread (...), and destructuring all work by repeatedly calling .next() like this behind the scenes, until done is true. Plain objects don't implement this protocol by default, which is exactly why for...of doesn't work on them directly.

Generators: an easier way to write iterators#

Writing an object with a manual .next() method is tedious. A generator function (declared with function*) lets you write iteration logic that reads like a normal loop, using yield to hand out one value at a time and pause.

JavaScript
function* countTo(max) {
  for (let i = 1; i <= max; i++) {
    yield i;
  }
}

for (const n of countTo(3)) {
  console.log(n);
}
// 1, 2, 3

Calling a generator function doesn't run its body immediately — it returns a generator object (itself iterable). Each call to .next() runs the function until the next yield, then pauses there until the next call.

JavaScript
function* idGenerator() {
  let id = 1;
  while (true) {          // an infinite generator is fine — it only runs on demand
    yield id;
    id++;
  }
}

const gen = idGenerator();
gen.next().value; // 1
gen.next().value; // 2
gen.next().value; // 3

Note

Generators are a niche tool day-to-day, but they're worth recognizing — they're what powers custom iteration logic, and they're the historical foundation async/await was designed on top of.

Map: key/value pairs done right#

A plain object is often used as a key/value store, but it has real limitations: keys are always coerced to strings, and it inherits properties from Object.prototype which can occasionally cause collisions. Map fixes both.

JavaScript
const scores = new Map();

scores.set("Ava", 95);
scores.set("Noah", 88);
scores.set(42, "the answer"); // any value can be a key, not just strings

scores.get("Ava");  // 95
scores.has("Noah"); // true
scores.size;         // 3
scores.delete("Noah");

for (const [name, score] of scores) {
  console.log(name, score);
}

Use Map over a plain object whenever keys aren't known ahead of time, aren't always strings, or when you need a reliable .size and guaranteed insertion order.

Set: unique values#

A Set stores unique values — adding a duplicate is silently ignored. The single most common use case: deduplicating an array in one line.

JavaScript
const numbers = [1, 2, 2, 3, 3, 3];
const unique = [...new Set(numbers)];
// [1, 2, 3]

const tags = new Set();
tags.add("js");
tags.add("js"); // ignored — already present
tags.size;       // 1
tags.has("js");  // true

WeakMap and WeakSet, briefly#

WeakMap and WeakSet work like their non-weak counterparts, but only accept objects as keys/values, and hold those references weakly — meaning they don't prevent the object from being garbage collected once nothing else references it (see Memory & Performance). They're a niche tool, mainly used to attach metadata to an object without causing a memory leak.

Symbols, briefly#

A Symbol is a primitive value that's guaranteed unique — even two symbols created with the same description are never equal. They're most often used as object property keys that are guaranteed not to collide with any string key, including ones added later by other code.

JavaScript
const id = Symbol("id");
const user = { name: "Ava", [id]: 12345 };

user[id]; // 12345
Symbol("id") === Symbol("id"); // false — always unique

Key Takeaways

  • for...of, spread, and destructuring all rely on the iterable protocol (Symbol.iterator) under the hood.
  • Generator functions (function*/yield) are an easy way to write custom, pausable iteration logic.
  • Map is a key/value store that accepts any key type and preserves insertion order — often a better fit than a plain object.
  • Set stores unique values and is the standard one-liner for deduplicating an array.
