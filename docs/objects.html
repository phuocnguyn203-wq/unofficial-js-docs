Working with Data

Objects Deep Dive

Objects are JavaScript's core way of grouping related data and behavior together under one value — key/value pairs, where a value can be anything, including another object or a function.

Object literals#

JavaScript
const user = {
  name: "Ava",
  age: 29,
  isAdmin: false,
  address: {
    city: "Tokyo",
    country: "Japan",
  },
};

Accessing and updating properties#

Two ways to read or write a property: dot notation and bracket notation.

JavaScript
user.name;          // "Ava" — dot notation, most common
user["name"];       // "Ava" — bracket notation, needed for dynamic keys

user.age = 30;       // update
user.email = "ava@example.com"; // add a brand-new property

const key = "age";
user[key];           // 30 — bracket notation lets the key be a variable

Dot notation is more readable and is the default choice. Bracket notation is required whenever the key is stored in a variable, computed, or contains characters that aren't valid in an identifier (like a space or a hyphen).

Shorthand syntax#

Two conveniences added in ES6 that you'll see everywhere:

JavaScript
const name = "Ava";
const age = 29;

// Property shorthand: when the variable name matches the key name
const user = { name, age }; // same as { name: name, age: age }

// Method shorthand: skip `function` for methods
const calculator = {
  add(a, b) {       // same as add: function(a, b) { ... }
    return a + b;
  },
};

Computed property names#

Wrap an expression in [ ] as a key to compute it dynamically at creation time:

JavaScript
const field = "email";
const user = {
  name: "Ava",
  [field]: "ava@example.com", // key becomes "email"
};
// { name: "Ava", email: "ava@example.com" }

Object.keys, values, and entries#

Three static methods for turning an object into an array you can loop over with familiar array methods:

JavaScript
const user = { name: "Ava", age: 29, admin: true };

Object.keys(user);    // ["name", "age", "admin"]
Object.values(user);  // ["Ava", 29, true]
Object.entries(user); // [["name", "Ava"], ["age", 29], ["admin", true]]

// entries() pairs nicely with destructuring in a for...of loop
for (const [key, value] of Object.entries(user)) {
  console.log(`${key}: ${value}`);
}

Checking if a property exists#

JavaScript
const user = { name: "Ava", age: undefined };

"name" in user;               // true
"age" in user;                // true — the key exists, even though its value is undefined
"email" in user;               // false

user.hasOwnProperty("name");   // true — ignores inherited properties (see Prototypes)
user.age !== undefined;        // false — unreliable if a value can legitimately be undefined

Prefer in or hasOwnProperty over checking !== undefined when you specifically care whether the key exists, not just whether it currently holds a value.

Copying objects#

Assigning an object to a new variable copies the reference, not the object (see Variables & Data Types). To actually copy the properties, use the spread operator or Object.assign:

JavaScript
const original = { name: "Ava", age: 29 };

const copy = { ...original };     // spread — the modern, common way
const copy2 = Object.assign({}, original); // older, equivalent approach

copy.name = "Nora";
original.name; // "Ava" — unaffected, copy is a separate object

Warning

Both approaches only make a shallow copy — nested objects are still shared by reference. If original.address is an object, copy.address points at the exact same nested object. For a true deep copy, you'd copy each nested level yourself, use structuredClone(original) (available in modern browsers), or a library.

JavaScript
const original = { name: "Ava", address: { city: "Tokyo" } };
const copy = { ...original };

copy.address.city = "Osaka";
original.address.city; // "Osaka" — changed! address was shared, not copied

Merging objects#

Spreading multiple objects together merges them, with later keys overwriting earlier ones:

JavaScript
const defaults = { theme: "light", fontSize: 14 };
const userPrefs = { fontSize: 18 };

const settings = { ...defaults, ...userPrefs };
// { theme: "light", fontSize: 18 }

Deleting a property#

JavaScript
const user = { name: "Ava", temp: "remove me" };
delete user.temp;
user; // { name: "Ava" }

Key Takeaways

  • Dot notation is the default; bracket notation is required for dynamic or non-identifier keys.
  • Property/method shorthand and computed property names are standard ES6+ conveniences.
  • Object.keys(), .values(), and .entries() turn an object into an array for iteration.
  • Spreading ({ ...obj }) or Object.assign copies an object, but only shallowly — nested objects are still shared.
