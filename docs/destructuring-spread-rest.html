Working with Data

Destructuring, Spread & Rest

Three closely related ES6 features that all revolve around one goal: pulling data apart or putting it together with less boilerplate. They show up constantly in modern JavaScript, including in most of the code examples elsewhere in this guide.

Array destructuring#

Unpack array values into named variables, positionally.

JavaScript
const coordinates = [10, 20, 30];
const [x, y, z] = coordinates;
// x = 10, y = 20, z = 30

const [first, , third] = coordinates; // skip an item with an empty slot
// first = 10, third = 30

const [a = 100, b = 100] = [5];
// a = 5 (provided), b = 100 (default, since [5] has no second item)

A famous one-liner this enables: swapping two variables without a temporary one.

JavaScript
let m = 1;
let n = 2;
[m, n] = [n, m];
// m = 2, n = 1

Object destructuring#

Unpack object properties into variables, by name instead of position.

JavaScript
const user = { name: "Ava", age: 29, city: "Tokyo" };

const { name, age } = user;
// name = "Ava", age = 29

const { name: fullName } = user;    // rename while destructuring
// fullName = "Ava"

const { country = "Unknown" } = user; // default value for a missing key
// country = "Unknown"

Destructuring can also go deep into nested objects in a single expression:

JavaScript
const response = {
  data: {
    user: { name: "Ava", role: "admin" },
  },
};

const { data: { user: { name, role } } } = response;
// name = "Ava", role = "admin"

Destructuring in function parameters#

This is one of the most common places you'll see destructuring — it turns a "big options object" into named, self-documenting parameters.

JavaScript
function createUser({ name, age, isAdmin = false }) {
  console.log(`${name}, age ${age}, admin: ${isAdmin}`);
}

createUser({ name: "Ava", age: 29 });
// "Ava, age 29, admin: false"

Callers don't need to remember argument order, and the function signature itself documents exactly what fields it expects.

The spread operator (...)#

... expands an iterable (array, string) or an object's own properties into individual elements. You've already seen it copy and merge objects in Objects Deep Dive — it works the same way for arrays.

JavaScript
const nums = [1, 2, 3];
const moreNums = [...nums, 4, 5]; // [1, 2, 3, 4, 5] — a new array

const combined = [...nums, ...[4, 5]]; // merge two arrays

Math.max(...nums); // 3 — spread an array into individual function arguments
JavaScript
const defaults = { theme: "dark" };
const overrides = { fontSize: 16 };
const settings = { ...defaults, ...overrides }; // { theme: "dark", fontSize: 16 }

The rest pattern (...)#

Confusingly, the exact same ... syntax means the opposite thing depending on where it appears: instead of expanding values, it gathers the leftover ones into a new array or object.

JavaScript
const [first, ...rest] = [1, 2, 3, 4];
// first = 1, rest = [2, 3, 4]

const { name, ...otherFields } = { name: "Ava", age: 29, city: "Tokyo" };
// name = "Ava", otherFields = { age: 29, city: "Tokyo" }

The rule of thumb: on the right side of an assignment (or in a function call), ... is spread — it expands. On the left side (destructuring) or in a function's parameter list, ... is rest — it gathers. Rest parameters in function definitions work exactly like the rest pattern here — see Functions for that use case.

Tip

Spread and rest use identical syntax on purpose — they're the same underlying idea (a list of values ↔ many individual values) applied in opposite directions. Once that clicks, the syntax stops feeling like two separate features to memorize.

Key Takeaways

  • Array destructuring unpacks by position; object destructuring unpacks by key name — both support defaults and renaming.
  • Destructuring function parameters turns an options object into clear, self-documenting named parameters.
  • Spread (...) expands an array/object into individual values — used to copy, merge, or pass many arguments at once.
  • Rest (...) gathers remaining values into a new array or object — the mirror image of spread.
