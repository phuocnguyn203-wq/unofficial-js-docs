Language Basics

Operators & Expressions

An expression is any piece of code that produces a value. Operators are how you combine, compare, and transform those values. This page is a tour of the operators you'll use every day.

Arithmetic operators#

JavaScript
10 + 3;   // 13  addition
10 - 3;   // 7   subtraction
10 * 3;   // 30  multiplication
10 / 3;   // 3.3333333333333335  division
10 % 3;   // 1   remainder (modulo)
10 ** 3;  // 1000 exponent (10 to the power of 3)

let count = 5;
count++;  // count is now 6 (post-increment)
count--;  // count is now 5 again (post-decrement)

Note

JavaScript has only one number type, so division always uses floating-point math. 10 / 3 doesn't truncate like it does in some languages — it gives you the full decimal.

Assignment operators#

Beyond the basic =, JavaScript has shorthand operators that combine an operation with assignment:

JavaScript
let total = 10;
total += 5;   // same as: total = total + 5   -> 15
total -= 3;   // total = total - 3            -> 12
total *= 2;   // total = total * 2            -> 24
total /= 4;   // total = total / 4            -> 6

let items = null;
items ??= [];  // assign [] only if items is null or undefined

Comparison operators#

JavaScript
5 > 3;    // true
5 < 3;    // false
5 >= 5;   // true
5 <= 4;   // false

5 === "5"; // false — strict equality, checks type too
5 == "5";  // true  — loose equality, coerces types
5 !== "5"; // true

See Variables & Data Types for why === is almost always the better choice over ==.

Logical operators & truthy/falsy values#

Before logical operators make sense, you need to know that every value in JavaScript is either truthy or falsy when used in a boolean context (like an if condition).

Falsy values (only these six)Everything else is truthy
false, 0, "", null, undefined, NaN"0", "false", [], {}, any non-empty string, any object
JavaScript
Boolean(0);        // false
Boolean("");       // false
Boolean("0");       // true  — a non-empty string is truthy, even "0"
Boolean([]);        // true  — an empty array is still an object, and truthy
Boolean(null);       // false

With that in mind, the three logical operators are:

JavaScript
true && false;   // false — AND: true only if both sides are true
true || false;   // true  — OR: true if either side is true
!true;           // false — NOT: flips the boolean

&& and || don't just produce true/false though — they return one of their original operands, thanks to short-circuiting:

JavaScript
// && returns the first falsy value, or the last value if all are truthy
"" && "hello";        // "" (stops at the first falsy value)
"hi" && "hello";      // "hello" (both truthy, returns the last one)

// || returns the first truthy value, or the last value if all are falsy
null || "default";    // "default"
"set" || "default";   // "set"

This is why you'll often see patterns like options || {} (fall back to an empty object) or user && user.name (only read .name if user exists).

The ternary operator#

A compact one-line if/else that produces a value:

JavaScript
const age = 20;
const status = age >= 18 ? "adult" : "minor";
console.log(status); // "adult"

Read it as: condition ? valueIfTrue : valueIfFalse. It's great for a simple either/or value — for anything more complex, a full if/else (see Control Flow) is more readable.

Nullish coalescing (??) and optional chaining (?.)#

Two operators added in ES2020 solve a very common annoyance: safely reading values that might be null or undefined.

JavaScript
// ?? only falls back on null/undefined — NOT on other falsy values like 0 or ""
const votes = 0;
votes || 10;   // 10   — oops, 0 is falsy, so this "falls back" unintentionally
votes ?? 10;   // 0    — 0 is neither null nor undefined, so it's kept

// ?. reads a property, but short-circuits to undefined instead of throwing
const user = { profile: null };
user.profile.bio;    // TypeError: Cannot read properties of null
user.profile?.bio;   // undefined — no error
user?.profile?.bio;  // undefined — safe even if `user` itself were missing

?? is the fix for ||'s biggest flaw: telling apart "no value" from "a real, falsy value" like 0 or "".

A note on operator precedence#

Just like in math, some operators bind tighter than others (* before +, for example). You don't need to memorize the full table — when in doubt, use parentheses to make the order explicit:

JavaScript
2 + 3 * 4;     // 14 — multiplication happens first
(2 + 3) * 4;   // 20 — parentheses force addition first

Key Takeaways

  • Only six values are falsy: false, 0, "", null, undefined, NaN — everything else is truthy.
  • && and || return one of their operands, not just true/false — this powers common default-value patterns.
  • ?? is like || but only triggers on null/undefined, not other falsy values.
  • ?. safely reads nested properties without throwing when something in the chain is missing.
  • When operator precedence gets confusing, add parentheses — clarity beats cleverness.
