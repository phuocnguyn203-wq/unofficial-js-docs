Language Basics

Control Flow

Control flow is how a program decides what to run, and how many times. JavaScript's tools for this — conditionals and loops — will look familiar if you've used almost any other C-like language.

if / else#

JavaScript
const hour = 14;

if (hour < 12) {
  console.log("Good morning");
} else if (hour < 18) {
  console.log("Good afternoon");
} else {
  console.log("Good evening");
}
// "Good afternoon"

Only the first condition that evaluates to truthy runs its block; every other branch is skipped. Braces are technically optional for single-statement blocks, but keep them — omitting them is a common source of bugs when a line gets added later.

switch#

switch compares one value against several possibilities using strict equality (===). It's a good fit when you have many exact matches to check against a single value.

JavaScript
const day = "Tue";

switch (day) {
  case "Mon":
  case "Tue":
  case "Wed":
  case "Thu":
  case "Fri":
    console.log("Weekday");
    break;
  case "Sat":
  case "Sun":
    console.log("Weekend");
    break;
  default:
    console.log("Not a valid day");
}
// "Weekday"

Warning

Forgetting break lets execution "fall through" into the next case. This is occasionally intentional (like "Mon" through "Fri" above sharing one outcome), but it's an easy mistake to make by accident. Always add break unless you deliberately want the fall-through.

The for loop#

The classic loop: initialize a counter, check a condition, and run an update after each pass.

JavaScript
for (let i = 0; i < 5; i++) {
  console.log(i);
}
// 0, 1, 2, 3, 4

The three parts are initialization; condition; update. The loop runs while the condition is truthy, running the update after every pass through the body.

while and do...while#

JavaScript
let n = 0;
while (n < 3) {
  console.log(n);
  n++;
}
// 0, 1, 2

while checks its condition before each pass — if the condition starts out false, the body never runs. do...while checks after, so the body always runs at least once:

JavaScript
let attempts = 0;
do {
  console.log("attempt", attempts);
  attempts++;
} while (attempts < 3);
// runs even if the condition were already false

for...of — looping over values#

for...of loops over the values of any iterable — arrays, strings, Maps, Sets, and more (see Iterators & Generators for what makes something "iterable"). It's the modern go-to for looping over an array's contents.

JavaScript
const fruits = ["apple", "banana", "cherry"];

for (const fruit of fruits) {
  console.log(fruit);
}
// "apple", "banana", "cherry"

for (const char of "hi") {
  console.log(char);
}
// "h", "i"

for...in — looping over keys#

for...in loops over the enumerable property names of an object. It's meant for plain objects, not arrays:

JavaScript
const user = { name: "Ava", role: "admin" };

for (const key in user) {
  console.log(key, "=", user[key]);
}
// "name = Ava"
// "role = admin"

Tip

Use for...of for arrays and other iterables, and for...in (or better, Object.keys()/Object.entries() — see Objects Deep Dive) for plain objects. Using for...in on an array technically works but can produce surprising results and is best avoided.

break and continue#

break exits a loop immediately. continue skips the rest of the current pass and moves to the next one.

JavaScript
for (let i = 0; i < 10; i++) {
  if (i === 5) break;       // stop the loop entirely
  if (i % 2 === 0) continue; // skip even numbers
  console.log(i);
}
// 1, 3

Choosing the right loop#

SituationUse
You have an array (or string, Map, Set) and want each valuefor...of, or an array method like .forEach()
You need the index too, or a custom stepclassic for
You're looping over a plain object's keysfor...in, or Object.keys()/.entries()
You don't know how many iterations you'll need up frontwhile
The loop body must run at least oncedo...while

Key Takeaways

  • if/else if/else runs the first truthy branch; switch compares one value against several exact cases.
  • for, while, and do...while are the general-purpose loops; do...while always runs its body at least once.
  • for...of iterates values (arrays, strings, Maps, Sets); for...in iterates an object's keys.
  • break exits a loop entirely; continue skips to the next iteration.
